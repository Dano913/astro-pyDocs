El control flow o control de flujo se refiere al orden en que se ejecutan las instrucciones en un programa o c√≥digo. El control flow determina qu√© instrucciones se ejecutan y en qu√© orden lo hacen y nos permitir√° controlar los posibles errores. Existen varias **estructuras de control flow**:

## ESTRUCTURA DE if
---
**if** es una estructura de control de flujo que se utiliza para tomar decisiones basadas en una condici√≥n. La sintaxis b√°sica de la funci√≥n **if** es la siguiente.

üìå**IMPORTANTE:** acordarse de poner los dos puntos : y a continuaci√≥n hacer una indentaci√≥n o sangr√≠a.

```python
if condici√≥n:
    `Secci√≥n de c√≥digo que se ejecuta si la condici√≥n es verdadera` 
```

- Si la condici√≥n **if** se eval√∫a como verdadera **True**, entonces se ejecuta el c√≥digo indentado que sigue a la sentencia.

- Si la expresi√≥n se eval√∫a como falsa **False**, entonces el c√≥digo indentado que sigue a la sentencia **if** se salta y el programa ejecuta la siguiente l√≠nea de c√≥digo que est√° indentada al mismo nivel que la sentencia **if**.

Veamos algunos ejemplos

```python
# Vamos a usar print("La condici√≥n se cumple") si la condici√≥n se cumple:
# Primera condici√≥n 5 es mayor que 2:
if 5 > 2:
	print("La condici√≥n se cumple")

# Vamos a intentarlo ahora con una condici√≥n que no se cumpla. 5 es menor que 2:

if 5 < 2:
	print("La condici√≥n se cumple otra vez")
# Output esperado: La condici√≥n se cumple

# Al no cumplirse la segunda condici√≥n, no se ejecutar√° esta secci√≥n del c√≥digo pero
# no provocaremos un error: Control de flujo.
```

Vamos a ver otro ejemplo

```python
lecciones_completadas = 3
print(f"Hemos completado estas lecciones {lecciones_completadas}")
# Output esperado:
# Hemos completado estas lecciones 3

# Completamos varias lecciones en un d√≠a, suponemos que 5:
lecciones_dia = 5
print(f"Hoy hemos completado estas lecciones {lecciones_dia}")
# Output esperado: Hoy hemos completado estas lecciones 5

# Definimos nuestra condicci√≥n. En este caso comprobamos si hoy hemos completado alguna lecci√≥n:
if lecciones_dia > 0:
	lecciones_completadas += lecciones_dia

# Comprobamos que se ha ejecutado
print(f"Las lecciones completadas ahora son: {lecciones_completadas}")

# Output esperado: Las lecciones completadas ahora son: 8
```

La estructura if en Python permite ejecutar c√≥digo basado en condiciones l√≥gicas. Es esencial para la toma de decisiones en un programa, ya que determina qu√© bloques de c√≥digo se ejecutan seg√∫n si las condiciones son verdaderas o falsas.

## Condicionales: if ... else
---
La estructura if ... else es otra forma de controlar el flujo del programa basado en una condici√≥n. Su sintaxis b√°sica es la siguiente:

```python
if condici√≥n:
    # Secci√≥n de c√≥digo que se ejecuta si la condici√≥n es verdadera
else:
    # Secci√≥n de c√≥digo que se ejecuta si la condici√≥n es falsa
```

üìå **IMPORTANTE**: Recordad usar los dos puntos : despu√©s de la condici√≥n y, a continuaci√≥n, hacer una indentaci√≥n o sangr√≠a tanto para el bloque **if** como para el bloque **else**.

‚óè Si la condici√≥n **if** se eval√∫a como verdadera **(True)**, entonces se ejecuta el c√≥digo indentado que sigue a la sentencia **if**, y el bloque **else** se ignora.

‚óè Si la expresi√≥n se eval√∫a como falsa **(False)**, entonces el c√≥digo indentado que sigue a la sentencia **else** se ejecuta, y el bloque **if** se salta.

Veamos algunos ejemplos

```python
# Definimos las lecciones completadas por ahora
lecciones_completadas = 3

# Completamos varias lecciones en un d√≠a:
lecciones_dia = 5

# Definimos nuestra condicci√≥n.
if lecciones_dia > 0:
	lecciones_completadas += lecciones_dia

else:
	print(f"Hoy no hemos completado ninguna lecci√≥n")

# Comprobamos que se ha ejecutado
print(f"Las lecciones completadas ahora son:{lecciones_completadas}")

# Output esperado: Las lecciones completadas ahora son:8
```
```python
# Hacemos lo mismo pero sin completar ninguna lecci√≥n hoy

# Definimos las lecciones completadas por ahora

# Completamos varias lecciones en un d√≠a

if lecciones_dia > 0:
	lecciones_completadas += lecciones_dia

else:
	print(f'Hoy no hemos completado ninguna lecci√≥n')

# Comprobamos que se ha ejecutado
print(f"Las lecciones completadas ahora son:{lecciones_completadas}")

# Output esperado:
# Hoy no hemos completado ninguna lecci√≥n
# Las lecciones completadas ahora son:3
```

## Condicionales: if... elif ... else
---
La estructura if ... elif ... else es utilizada para tomar decisiones m√∫ltiples basadas en m√∫ltiples condiciones. Su sintaxis b√°sica es la siguiente:

```python
if condici√≥n_1:
    # Secci√≥n de c√≥digo que se ejecuta si la condici√≥n_1 es verdadera
elif condici√≥n_2:
    # Secci√≥n de c√≥digo que se ejecuta si la condici√≥n_1 es falsa y la condici√≥n_2 es verdadera
else:
    # Secci√≥n de c√≥digo que se ejecuta si ninguna de las condiciones anteriores es verdadera
```

üìå **IMPORTANTE**: Recordad usar los dos puntos : despu√©s de cada condici√≥n **(if, elif y else)**, y a continuaci√≥n, hacer una indentaci√≥n o sangr√≠a tanto para el bloque correspondiente como para el bloque siguiente.

‚óè Si la condici√≥n **if** se eval√∫a como verdadera **(True)**, entonces se ejecuta el c√≥digo indentado que sigue a la sentencia **if**, y los bloques **elif** y **else** se ignoran.

‚óè Si la condici√≥n **if** es falsa y la condici√≥n **elif** se eval√∫a como verdadera **(True)**, entonces se ejecuta el c√≥digo indentado que sigue a la sentencia **elif**, y el bloque **else** se ignora.

‚óè Si ninguna de las condiciones anteriores es verdadera, entonces se ejecuta el c√≥digo indentado que sigue a la sentencia **else**.

Vamos a ver un ejemplo:

```python
# Vamos a comprobar si hemos completado m√°s de 5 lecciones hoy

# Definimos las lecciones completadas por ahora
lecciones_completadas = 3

# Completamos varias lecciones en un d√≠a
lecciones_dia = 7

# Definimos nuestra condicci√≥n
if lecciones_dia == 5:
	lecciones_completadas += lecciones_dia
	print(f"Hemos completado 5 lecciones hoy")
elif lecciones_dia > 5:
	lecciones_completadas += lecciones_dia
	print(f"Hemos completado m√°s de 5 lecciones hoy")
else:
	print(f"Hoy hemos completado menos de 5 lecciones")

# Comprobamos que se ha ejecutado:
print(f"Hoy hemos completado estas lecciones: {lecciones_dia}")
print(f"Las lecciones completadas hasta ahora son: {lecciones_completadas}")

# Output esperado:
# Hemos completado m√°s de 5 lecciones hoy
# Hoy hemos completado estas lecciones: 7
# Las lecciones completadas hasta ahora son: 10
```

Los condicionales **if ... elif ... else** permiten evaluar m√∫ltiples condiciones en secuencia. El bloque **if** se eval√∫a primero; si la condici√≥n es verdadera, su c√≥digo se ejecuta. Si no, se eval√∫an las condiciones de los bloques **elif**. Si ninguna de las condiciones anteriores es verdadera, se ejecuta el bloque **else**, que es opcional.

## Condicionales: if anidados
---
Ocurre cuando incluimos una sentencia **if** dentro de otra sentencia **if**. Los **if** anidados nos permiten construir programas muy complejos que responden a diferentes condiciones.

Es importante tener en cuenta que el uso excesivo de **if** anidados puede hacer que el c√≥digo sea dif√≠cil de leer y mantener.

Es recomendable utilizarlos con moderaci√≥n y, en su lugar, utilizar t√©cnicas de programaci√≥n m√°s avanzadas, como las estructuras de datos y los algoritmos.

Vamos a ver algunos ejemplos:

```python
# Suponemos que un tema consta de 10 lecciones.

# Completamos varias lecciones en un d√≠a
lecciones_dia = 7

if lecciones_dia > 0:
	# primero sumamos las lecciones del d√≠a:
	lecciones_completadas += lecciones_dia
	if lecciones_completadas >= 10:
		print(f'¬°Hemos completado el tema!')
	else:
		print(f'A√∫n no hemos completado el tema')
else:
	print(f'Hoy no hemos completado ninguna lecci√≥n...')

print(f'Hoy hemos completado estas lecciones: {lecciones_dia}')
print(f'Las lecciones completadas hasta ahora son: {lecciones_completadas}')

# Output esperado:
# ¬°Hemos completado el tema!
# Hoy hemos completado estas lecciones: 7
# Las lecciones completadas hasta ahora son: 10
```

Los condicionales **if** anidados permiten incluir una estructura **if ... else** dentro de otra. Aunque son √∫tiles para casos complejos, su uso excesivo puede reducir la legibilidad del c√≥digo, por lo que conviene utilizarlos con moderaci√≥n.

## Condicionales: if, and or
---
- El **and** lo usaremos cuando queramos que se cumplan **TODAS** las condiciones.

- El **or** lo usaremos cuando queramos que se cumplan **ALGUNA** de las condiciones.

Vamos a ver algunos ejemplos

```python
# Definimos dos nuevas variables
dia_semana1 = 'viernes'
hora1 = 17

# Usamos if con and y or para realizar diferentes comprobaciones basadas en el d√≠a de la semana y la hora

# Comprobamos si es viernes y si la hora es igual o mayor a 15
if dia_semana1 == 'viernes' and hora1 >= 15:
    print(f' Es viernes y ya es finde semana')

# Si no se cumple la condici√≥n anterior, comprobamos si es s√°bado o domingo
elif dia_semana1 == 'sabado' or dia_semana1 == 'domingo':
    print(f'Es finde semana')

# Si no se cumple ninguna de las condiciones anteriores, y si no es viernes, s√°bado ni domingo, pero la hora es mayor o igual a 18
elif dia_semana1 != 'viernes' and dia_semana1 !='sabado' and dia_semana1 !='domingo' and hora1 >= 18:
    print(f'Ya has trabajado bastante por hoy')

# Si ninguna de las condiciones anteriores se cumple
else:
    print(f'Hay que trabajar')

# Output esperado: Es viernes y ya es finde semana
```

## Condicionales: while
---
Los bucles **while** son una estructura de control que permite ejecutar un bloque de c√≥digo varias veces mientras se cumpla una determinada condici√≥n. El bucle **while** se repite continuamente mientras la condici√≥n especificada sea veradera. Si la condici√≥n es falsa, el bucle **while** termina y la ejecuci√≥n contin√∫a con el c√≥digo que sigue al bucle.

La condici√≥n en el bucle **while** puede ser cualquier expresi√≥n booleana, es decir, una condici√≥n que sea **True** o **False**. Usamos un **while** cuando no sabemos cu√°ntas veces vamos a tener que ejecutar el bucle.

La sintaxis b√°sica del bucle while en Python es la siguiente:

üìå **IMPORTANTE**: acordaos de poner los dos puntos `:` y a continuaci√≥n hacer una indentaci√≥n o sangr√≠a.

üìå **IMPORTANTE**: si la condici√≥n del bucle **while** nunca se vuelve falsa, el bucle se ejecutar√° infinitamente, lo que puede llevar a errores y problemas en el programa. Por lo tanto, es importante asegurarse de que la condici√≥n se vuelve falsa en alg√∫n momento.

```python
Vamos a ver varios ejemplos

# Supongamos que tenemos 23 d√≠as de vacaciones al a√±o:
dias_vacaciones = 23

# Mientras que los d√≠as de vacaciones sean mayores a 0, a√∫n nos quedan d√≠as.
while dias_vacaciones > 0:

		# Imprimimos el n√∫mero de d√≠as de vacaciones restantes.
    print(f"A√∫n nos quedan: {dias_vacaciones} d√≠as de vacaciones")

    # Reducimos el contador de d√≠as de vacaciones en 1 en cada iteraci√≥n.
    dias_vacaciones -= 1
```

Vamos a juntarlo con una condici√≥n **if**

```python
# Supongamos que tenemos 23 d√≠as de vacaciones al a√±o:
dias_vacaciones = 23

# Mientras que los d√≠as de vacaciones sean mayores a 0, todav√≠a nos quedan d√≠as.
while dias_vacaciones > 0:
    # Imprimimos el n√∫mero de d√≠as de vacaciones restantes.
    print(f'A√∫n nos quedan: {dias_vacaciones} d√≠as de vacaciones')

    # Reducimos el contador de d√≠as de vacaciones en 1 en cada iteraci√≥n.
    dias_vacaciones -= 1

    # Si los d√≠as de vacaciones llegan a 0, mostramos un mensaje indicando que se han agotado los d√≠as.
    if dias_vacaciones == 0:
        print(f'Se nos han acabado los d√≠as')
```

El bucle while en Python ejecuta un bloque de c√≥digo de manera repetitiva mientras una condici√≥n sea verdadera. Es √∫til para situaciones en las que no sabemos de antemano cu√°ntas veces se debe repetir el ciclo, ya que sigue ejecut√°ndose hasta que la condici√≥n se vuelva falsa.
