La programaci√≥n orientada ( Object Oriented Programing )a objetos nos permite estructurar los programas de tal manera que podamos agrupar ciertas propiedades y comportamientos en objetos individuales.

Un objeto individual es una replica de un objeto del mundo real, como un mueble, que tiene propiedades, como el color, la forma, el n√∫mero de puertas y comportamientos propios, abrir puertas.

La programaci√≥n orientada a objetos modela las entidades del mundo real como objetos de software que tienen algunos datos asociados y pueden realizar ciertas funciones.

## Conceptos b√°sicos:
---
- **Clase**: es un esquema o plantilla a partir de la cual se crear√°n objetos. Define las caracter√≠sticas comunes que tendr√°n los objetos que derivan de ella. Es como un modelo que establece qu√© atributos (caracter√≠sticas) y m√©todos (comportamientos) tendr√° un objeto.
Podemos tener una clase llamada Empleado. Esta clase podr√≠a definir que todos los empleados tienen un nombre, una posici√≥n y un salario.

- **Atributos**: son las caracter√≠sticas que tendr√° cada objeto de una clase.
En el ejemplo de la clase Empleado, los atributos son nombre, posicion y salario. Cada objeto (empleado) tendr√° valores espec√≠ficos para estos atributos.

- **M√©todos**: on las funciones que puede realizar una clase. Representan las acciones que un objeto puede llevar a cabo o las operaciones que pueden realizarse sobre √©l. Los m√©todos permiten a los objetos interactuar con sus propios atributos o realizar acciones relacionadas con ellos.
La clase Empleado puede tener un m√©todo que imprima los detalles de un empleado.

- **Objeto** o **Instancia** es una entidad individual creada a partir de una clase. Cada objeto tendr√° sus propios valores para los atributos definidos en la clase.
Por ejemplo, un objeto de la clase Empleado ser√≠a un empleado en particular, como "Ana", con su propio nombre, posici√≥n y salario.

üìå Por convenci√≥n, para que sepamos que se est√°n trabajando con Clases, estas ser√°n nombradas con la primera letra en may√∫sculas.

## Atributos y m√©todo constructor
---
Lo primero que hacemos es definir las propiedades de nuestra clase, es decir los **atributos**. Todos los **atributos** deben estar definidos en un m√©todo llamado m√©todo **constructor**

El m√©todo constructor se encuentra dentro de la clase y tiene por par√°metros las caracter√≠sticas que tendr√° nuestra clase siguiendo esta sintaxis:

```python
class NombreClase:

	def __init__ (self, atributo1, atributo2, atributo3)

# Donde de __init__ es el m√©todo constructor
# self es el primer par√°metro
# los atributos son las propiedades que queremos que tenga nuestra clase
```

Vemos un ejemplo con una clase llamada **Empleado**:

```python
class Empleado:
    """
    Clase que representa a un empleado en una empresa.
    """

    def __init__(self, nombre, edad, posicion, salario):
        """
        Constructor de la clase Empleado.

        Args:
            nombre (str): El nombre del empleado.
            edad (int): La edad del empleado.
            posicion (str): La posici√≥n del empleado.
            salario (float): El salario del empleado.
        """
        # Creamos atributos y asignamos el valor de los par√°metros del m√©todo constructor
        self.nombre = nombre
        self.edad = edad
        self.posicion = posicion
        self.salario = salario
```

## M√©todos de una clase
---
Una vez tengamos el m√©todo constructor definimos las acciones que queremos que tenga nuestra clase, es decir, sus m√©todos propios. Los m√©todos aparecen a continuaci√≥n del m√©todo constructor. Por ejemplo un m√©todo que sirva para subir el salario seg√∫n un porcentaje:

```python
def subir_salario(self, porcentaje):
        """
        M√©todo para aumentar el salario del empleado en un cierto porcentaje.

        Args:
            porcentaje (float): El porcentaje de aumento del salario.
        """
        # Calcula el nuevo salario aumentado por el porcentaje dado
        self.salario *= (1 + porcentaje / 100)
        # Redondea el salario a dos decimales
        self.salario = round(self.salario, 2)
        # Muestra un mensaje indicando que el salario ha subido y cu√°l es el nuevo salario
        print(f"¬°El salario de {self.nombre} ha subido! Nuevo salario: {self.salario:.2f}")
```

Para usar este m√©todo en nuestro objeto **juan**, tendremos que llamar al m√©todo, donde:

**juan** es el objeto que hemos instanciado previamente.

**subir_salario()** es el m√©todo que acabamos de crear.

**10** es el porcentaje del salario que va a subirse.

```python

# Output esperado:
# ¬°El salario de Juan ha subido! Nuevo salario: 55000.00
```

## Herencia
---
La herencia dentro de programaci√≥n nos permite crear nuevas clases que tengan atributos y m√©todos de otras clases que ya hemos definido anteriormente

La primera clase que creamos ser√° la clase **madre** o **superclase** o ***parent***. La clase que hereda, es decir, que recibe atributos y m√©todos de la primera ser√° la clase **hija** o **subclase** o ***child***.

En las clases hijas podemos cambiar los atributos y m√©todos heredadas y tambi√©n incorporar otros nuevos que ser√°n exclusivos de ellas mismas y que no tendr√° la clase madre.

Vamos a ver un ejemplo con productos. Podemos tener una clase madre que se lllame **Producto**, que tenga un m√©todo constructor con sus atributos y un m√©todo **impuestos** para calcular el precio con IVA**:**

```python
class Producto:
    """
    Clase que representa un producto para la venta.
    """

    def __init__(self, nombre, precio, plataforma_venta, IVA):
        """
        Constructor de la clase Producto.

        Args:
            nombre (str): El nombre del producto.
            precio (float): El precio del producto.
            plataforma_venta (str): La plataforma de venta del producto.
        """
        self.nombre = nombre
        self.precio = precio
        self.plataforma_venta = plataforma_venta
        self.IVA = IVA

    def impuestos(self):
        """
        M√©todo para calcular los impuestos sobre el precio del producto.

        Returns:
            float: La cantidad de impuestos a pagar.
        """
        return self.precio * self.IVA
```

Podemos tener una clase hija de la clase **Producto**, por ejemplo la clase **Electrodomestico**. Esta clase va a heredar los atributos y los m√©todos de la clase madre y va a a√±adir dos atributos nuevos: **certificado_energetico** y un m√©todo propio que sea **evaluar_certificado**.

```python
class Electrodomestico(Producto):
    """
    Clase que representa un electrodom√©stico como un tipo de producto.
    """

    def __init__(self, nombre, precio, plataforma_venta, IVA, certificado_energetico):
        """
        Constructor de la clase Electrodomestico.

        Args:
            nombre (str): El nombre del electrodom√©stico.
            precio (float): El precio del electrodom√©stico.
            plataforma_venta (str): La plataforma de venta del electrodom√©stico.
            marca (str): La marca del electrodom√©stico.
            consumo_energetico (str): El nivel de consumo energ√©tico del electrodom√©stico.
        """
        # Llama al constructor de la clase madre
        super().__init__(nombre, precio, plataforma_venta, IVA)

        # A√±adimos los argumentos espec√≠ficos de la clase hija
        self. certificado_energetico = certificado_energetico

    def evaluar_certificado(self):

        if self.certificado_energetico == "F":
            print(f"El producto {self.nombre} tiene un certificado energ√©tico deficiente")
```

Podemos ponerlo en pr√°ctica instanciando un objeto de la clase **Electrodomestico**:

```python

print(lavadora.nombre)

# Output esperado:
#lavadora siemens
#El producto lavadora siemens tiene un certificado energ√©tico deficiente
```

## Polimorfismo
---
El polimorfismo es un concepto que se refiere a la capacidad de distintos objetos de una jerarqu√≠a de clases de responder a una misma acci√≥n de diferentes maneras, es decir, que podemos tener un m√©todo que se llame igual dentro de distintas clases pero que se comporte de una manera para cada una de esas clases.

Por ejemplo podemos tener el m√©todo **precio_iva**, de la clase madre **Producto**, que calcule el precio aplicando el IVA:

```python
class Producto:
        #...
        def precio_iva(self):
                return self.precio * (1 + self.IVA)
```

Y queremos que aquellos productos que sean **electrodom√©sticos** se calcule ese precio con IVA pero restandole 100. Lo ¬¥√∫nico que tendr√≠amos que hacer es crear un m√©todo **precio_iva** en la clase **Electrodomesticos** que contemple esta resta:

```python

        #...
        def precio_iva(self):
                return self.precio * (1 + self.IVA) - 100
```

El m√©todo **precio_iva** se llama de la misma manera en ambas clases pero tiene un comportamiento distinto.
